OpenClaw Windows 一键安装与卸载说明书

适用对象：不会自己装 Node.js、不会配 URL / Key、希望把 OpenClaw 和常用渠道一次配好，也希望后续能统一卸载干净的 Windows 用户。

本安装包覆盖模型提供商、渠道配置、插件补装、Skills 选择安装、安装摘要、辅助启动脚本，以及新增的统一风格卸载向导。

1. 这套安装包做了什么

  • 支持选择 npm 镜像源（淘宝镜像），国内用户安装速度大幅提升。
  • 调用 OpenClaw 官方 Windows 安装脚本，自动处理 Node.js 和 OpenClaw CLI。
  • 在安装过程中按步骤提示填写模型 API Key、Base URL、模型 ID。
  • 填写 API Key 后自动验证连通性，确认 Key 是否可用。
  • 可选配置 Feishu、QQ Bot、Telegram、Discord、Slack、LINE、WhatsApp。
  • 只有在用户勾选了某个渠道后，才会继续询问该渠道需要的 token、secret、webhook 路径或回调 URL。
  • 提供推荐 Skills 选择安装，并支持手动输入额外 skill slug。
  • 安装后生成摘要文件和常用的 .bat 辅助脚本。
  • 新增 uninstall-openclaw.bat，用于统一卸载服务、状态目录、工作目录、CLI 和安装器产物。

2. 使用方法

方式 A：交互式安装

  1. 双击 install-openclaw.bat。
  2. 按向导选择模型来源。
  3. 填写 API Key、Base URL、模型 ID 等信息。
  4. 勾选要接入的渠道。
  5. 为已勾选渠道填写凭据；暂时不配的可以跳过。
  6. 勾选要安装的 Skills。

方式 B：预设配置安装

  1. 复制 installer-config.example.json。
  2. 改名为 installer-config.local.json。
  3. 预先填好模型、渠道、Skills。
  4. 双击 install-openclaw.bat 直接安装。

方式 C：交互式卸载

  1. 双击 uninstall-openclaw.bat。
  2. 按向导选择是否先创建备份。
  3. 勾选要删除的内容：网关服务、~/.openclaw、工作目录、全局 CLI、安装器生成文件。
  4. 完成后查看 outputs/openclaw-uninstall-summary.txt。

3. 模型配置与 API Key 获取

模式需要填写说明网络要求
OpenAIapiKey, modelRef默认示例为 openai/gpt-5.4。需要科学上网
OpenRouterapiKey, modelRef默认示例为 openrouter/auto。需要科学上网
MoonshotapiKey, modelRef默认示例为 moonshot/kimi-k2.5。国内直连，无需梯子
自定义兼容 OpenAIbaseUrl, apiKey, providerId, customModelId适用于中转站、自建兼容接口或本地推理网关。取决于提供商
OpenAI 和 OpenRouter 的 API 服务器在海外，安装和使用过程中都需要科学上网（梯子/VPN）。如果你在国内且没有梯子，建议选择 Moonshot 或国内自定义提供商。

3.1 去哪获取 API Key

OpenAI

  1. 打开 platform.openai.com/api-keys
  2. 登录后点击 “Create new secret key”
  3. 复制生成的 sk-... 密钥

OpenRouter

  1. 打开 openrouter.ai/keys
  2. 登录后创建 Key
  3. 复制生成的密钥

Moonshot

  1. 打开 platform.moonshot.cn/console/api-keys
  2. 登录后新建 API Key
  3. 复制生成的密钥
安装向导填写 API Key 后会自动发送测试请求验证连通性。如果验证失败也可以继续安装，之后再修正。

4.1 飞书 / Lark

打开 open.feishu.cn/app，创建企业自建应用，在“凭证与基本信息”里复制 App ID 和 App Secret。

推荐 websocket 模式，不需要公网服务器。webhook 模式还需要 Verification Token 和 Encrypt Key。

4.2 QQ Bot

打开 q.qq.com，创建机器人应用，在“开发设置”里复制 AppID 和 AppSecret。

4.3 Telegram

在 Telegram 中搜索 @BotFather，发送 /newbot，按提示创建机器人并复制 token。

4.4 Discord

打开 discord.com/developers/applications，创建应用后进入 Bot 页面，重置并复制 Bot Token。

必须在 Bot 页面开启 Message Content Intent，否则机器人无法读取消息内容。

4.5 Slack

打开 api.slack.com/apps 创建应用。Socket 模式需要 xoxb-... 和 xapp-...；HTTP 模式需要 Signing Secret 和回调 URL。

Socket 模式不需要公网服务器，适合本地开发和内网部署。

4.6 LINE

打开 developers.line.biz/console/，创建 Messaging API Channel，生成 Access Token 并复制 Channel Secret。

LINE 必须使用 webhook 回调，需要公网可访问的 HTTPS 地址。

4.7 WhatsApp

WhatsApp 渠道不需要预先获取 token。安装器会在安装完成后自动运行登录命令，终端显示二维码后，用手机 WhatsApp 扫码即可。

4.8 策略说明

策略含义适用场景
pairing 配对用户需输入配对码才能对话推荐私聊，防止陌生人使用
allowlist 白名单仅白名单中的用户可使用企业内部或特定群组
open 开放所有人可直接使用公开服务或测试环境
disabled 关闭完全关闭该功能暂时不启用时选择
大多数情况下，私聊选 pairing、群聊选 allowlist 是最安全的默认选择。

5. 渠道回调 URL

如果选择了 Slack HTTP、LINE 或飞书 webhook 模式，安装器会询问公网网关地址，例如：

https://bot.example.com

安装摘要会自动拼出回调 URL：

  • Slack HTTP: https://bot.example.com/slack/events
  • LINE: https://bot.example.com/line/events
  • 飞书 webhook: https://bot.example.com/feishu/events

7. npm 镜像源选择

安装向导会询问你使用哪个 npm 镜像源。国内用户强烈建议选择淘宝镜像，安装速度可提升数倍。

选项地址适用场景
默认 (Default)registry.npmjs.org海外用户或已有科学上网
淘宝镜像 (China mirror)registry.npmmirror.com国内用户，无需梯子即可加速
安装器会在安装前自动设置 npm registry，安装完成后恢复你原来的源设置，不会影响你其他项目。
预设配置安装时，在 installer-config.local.json 的 openclaw.npmMirror 字段填入 https://registry.npmmirror.com 即可。留空则使用默认源。

9. 卸载时会做什么

  • 可选先运行 openclaw backup create 创建备份。
  • 优先调用 openclaw uninstall 卸载网关服务、状态目录、工作目录。
  • 如果你勾选删除 CLI，会继续执行 npm rm -g openclaw。
  • 如果检测不到 openclaw.cmd，卸载器会退回手动清理模式。
  • 安装器生成的 launchers/、installer-config.generated.json、安装摘要也可以一并清理。
如果当前工作目录不在 ~/.openclaw 下，卸载向导默认不会直接勾选删除，避免误删业务文件。

10. 常见问题

双击没反应

右键用 PowerShell 运行，或手动执行：

powershell -NoLogo -NoProfile -ExecutionPolicy Bypass -File .\install-openclaw.ps1

只想走卸载流程

执行：

powershell -NoLogo -NoProfile -ExecutionPolicy Bypass -File .\install-openclaw.ps1 -Uninstall

安装完找不到 openclaw

关掉当前终端，重新开 PowerShell，再试：

openclaw --version

API Key 验证失败

  • 检查 Key 是否复制完整，没有多余空格或截断。
  • 确认提供商账户有余额或有效订阅。
  • 自定义提供商确认 Base URL 格式正确，通常以 /v1 结尾。
  • 验证失败不影响安装，之后可以在 ~/.openclaw/.env 中修改。

Slack / LINE / 飞书 webhook 没有收到事件

  • 先确认公网回调 URL 可访问。
  • 确认平台控制台里的路径和安装摘要里生成的一致。
  • 再看 openclaw gateway status 和 openclaw logs --follow。

WhatsApp 一直没连上

重新执行：

openclaw channels login --channel whatsapp

自定义工作目录会被自动删除吗

交互式卸载里会单独确认；如果工作目录不在 ~/.openclaw 下，默认不会直接勾选删除。

11. 安全建议

不要把 .env、API Key、AppSecret、bot token 上传到公开仓库。打包发给别人时，仓库里只放示例配置，不放真实密钥。
