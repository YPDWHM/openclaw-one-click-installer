OpenClaw Windows 一键安装说明书

适用对象：不会自己装 Node.js、不会配 URL / Key、希望把 OpenClaw 和常用渠道一次配好的 Windows 用户。

本安装包覆盖模型提供商、渠道配置、插件补装、skills 选择安装、安装摘要和辅助启动脚本。

1. 这套安装包做了什么

  • 调用 OpenClaw 官方 Windows 安装脚本，自动处理 Node.js 和 OpenClaw CLI。
  • 在安装过程中按步骤提示填写模型 API Key、Base URL、模型 ID。
  • 填写 API Key 后自动验证连通性，确认 Key 是否可用。
  • 可选配置 Feishu、QQ Bot、Telegram、Discord、Slack、LINE、WhatsApp。
  • 只有在用户勾选了某个渠道后，才会继续询问该渠道需要的 token、secret、webhook 路径或回调 URL。
  • 提供推荐 skills 选择安装，并支持手动输入额外 skill slug。
  • 安装后生成摘要文件和几个常用的 .bat 辅助脚本。

2. 使用方法

方式 A：交互式

  1. 双击 install-openclaw.bat。
  2. 按向导选择模型来源。
  3. 填写 API Key、Base URL、模型 ID 等信息（向导会提示去哪获取）。
  4. 勾选要接入的渠道。
  5. 为已勾选渠道填写凭据；暂时不配的可以跳过。
  6. 勾选要安装的 skills。

方式 B：预设配置

  1. 复制 installer-config.example.json。
  2. 改名为 installer-config.local.json。
  3. 预先填好模型、渠道、skills。
  4. 双击 install-openclaw.bat 直接安装。

3. 模型配置与 API Key 获取

模式需要填写说明
OpenAIapiKey, modelRef默认示例是 openai/gpt-5.4。
OpenRouterapiKey, modelRef默认示例是 openrouter/auto。
MoonshotapiKey, modelRef默认示例是 moonshot/kimi-k2.5。
自定义兼容 OpenAIbaseUrl, apiKey, providerId, customModelId适用于中转站、自建兼容接口或本地推理网关。

3.1 去哪获取 API Key

OpenAI

  1. 打开 platform.openai.com/api-keys
  2. 登录后点击 "Create new secret key"
  3. 复制生成的 sk-... 开头的密钥

OpenRouter

  1. 打开 openrouter.ai/keys
  2. 登录后点击 "Create Key"
  3. 复制生成的密钥

Moonshot (Kimi)

  1. 打开 platform.moonshot.cn/console/api-keys
  2. 登录后点击 "新建 API Key"
  3. 复制生成的密钥

自定义兼容提供商

  1. 从你的提供商控制台获取 API Key
  2. 确认 Base URL 格式，通常以 /v1 结尾
  3. 如果后端报 content must be string，把 requiresStringContent 改成 true
  4. 如果工具调用会出错，把 supportsTools 改成 false
安装向导填写 API Key 后会自动发送测试请求验证连通性。如果验证失败也可以继续安装，之后再修正。

4.1 飞书 / Lark

  1. 打开 open.feishu.cn/app，登录后点击"创建企业自建应用"
  2. 进入应用详情页，在左侧菜单找到"凭证与基本信息"
  3. 复制 App ID 和 App Secret
  4. 在左侧菜单"事件与回调"中，开启"使用长连接接收事件"（即 websocket 模式）
  5. 在"权限管理"中添加所需权限（至少需要接收消息和发送消息的权限）
推荐使用 websocket 模式，不需要公网服务器。如果选择 webhook 模式，还需要在"事件与回调"页面获取 Verification Token 和 Encrypt Key。

4.2 QQ Bot

  1. 打开 q.qq.com，登录 QQ 开放平台
  2. 进入"应用管理"，创建一个机器人应用
  3. 在"开发设置"中找到并复制 AppID 和 AppSecret
安装器会自动补装 @openclaw/qqbot 插件。

4.3 Telegram

  1. 在 Telegram 中搜索 @BotFather 并打开对话
  2. 发送 /newbot，按提示输入机器人名称和用户名
  3. 创建成功后，BotFather 会返回一个 Bot Token，格式类似 123456789:ABCdefGHI...
  4. 复制这个 token，在安装向导中填入
首次私聊机器人通常需要 pairing（配对码验证）。可以在安装后通过 openclaw pairing list 查看配对码。

4.4 Discord

  1. 打开 discord.com/developers/applications，登录后点击 "New Application"
  2. 进入应用后，在左侧菜单点击 "Bot"
  3. 点击 "Reset Token" 生成新的 Bot Token 并复制
  4. 在同一页面下方，开启 Message Content Intent（必须，否则机器人收不到消息内容）
  5. 在左侧 "OAuth2 > URL Generator" 中，勾选 bot scope 和所需权限，生成邀请链接将 bot 添加到你的服务器
必须开启 Message Content Intent，否则机器人无法读取消息内容。

4.5 Slack

  1. 打开 api.slack.com/apps，点击 "Create New App" > "From scratch"
  2. 在 OAuth & Permissions 中添加 Bot Token Scopes（至少 chat:write、app_mentions:read）
  3. 点击 "Install to Workspace"，授权后复制 Bot User OAuth Token（xoxb-...）

Socket 模式（推荐）：

  1. 在 Basic Information > App-Level Tokens 中创建 token（scope: connections:write）
  2. 复制 xapp-... 开头的 App Token
  3. 在左侧 Socket Mode 页面开启 Socket Mode

HTTP 模式：

  1. 在 Basic Information > App Credentials 中复制 Signing Secret
  2. 在 Event Subscriptions 中填写回调 URL（安装摘要中会生成）
Socket 模式不需要公网服务器，适合本地开发和内网部署。

4.6 LINE

  1. 打开 developers.line.biz/console，登录后创建 Provider 和 Messaging API Channel
  2. 在 Channel 详情页的 Messaging API 标签下，点击 "Issue" 生成 Channel Access Token
  3. 在 Basic settings 标签下复制 Channel Secret
  4. 在 Messaging API 标签下的 Webhook URL 中填入安装摘要里生成的回调地址
LINE 必须使用 webhook 回调，需要公网可访问的 HTTPS 地址。

4.7 WhatsApp

  1. WhatsApp 渠道不需要预先获取 token
  2. 安装器会在安装完成后自动运行 openclaw channels login --channel whatsapp
  3. 终端会显示一个二维码，用手机 WhatsApp 扫码即可完成登录
如果扫码超时或断开，可以重新运行 openclaw channels login --channel whatsapp。

4.8 策略说明

安装向导中每个渠道都会让你选择"私聊策略"和"群聊策略"，含义如下：

策略含义适用场景
pairing 配对模式用户首次使用时需要输入配对码才能与机器人对话推荐用于私聊，防止陌生人随意使用
allowlist 白名单仅白名单中的用户或群组可以使用适合企业内部或特定群组使用
open 开放所有人都可以直接使用，无需验证适合公开服务或测试环境
disabled 关闭完全关闭该功能不需要私聊或群聊功能时选择
大多数情况下，私聊策略选 pairing、群聊策略选 allowlist 是最安全的默认选择。安装后可以通过 openclaw pairing list 查看和管理配对码。

5. 渠道回调 URL 怎么看

如果你选择了需要回调地址的模式（Slack HTTP、LINE、飞书 webhook），安装器会询问公网网关地址。例如：

https://bot.example.com

随后安装摘要里会自动帮你拼出：

  • Slack HTTP: https://bot.example.com/slack/events
  • LINE: https://bot.example.com/line/events
  • 飞书 webhook: https://bot.example.com/feishu/events

8. 常见问题

双击没反应

右键用 PowerShell 运行，或手动执行：

powershell -NoLogo -NoProfile -ExecutionPolicy Bypass -File .\install-openclaw.ps1

安装完找不到 openclaw

关掉当前终端，重新开 PowerShell，再试：

openclaw --version

API Key 验证失败

  • 检查 Key 是否复制完整（没有多余空格或截断）。
  • 确认提供商账户有余额或有效的订阅。
  • 如果是自定义提供商，确认 Base URL 格式正确（通常以 /v1 结尾）。
  • 验证失败不影响安装，可以之后在 ~/.openclaw/.env 中修改。

Slack / LINE / 飞书 webhook 没有收到事件

  • 先确认公网回调 URL 可访问。
  • 确认平台控制台里填的路径和安装摘要里生成的一致。
  • 再看 openclaw gateway status 和 openclaw logs --follow。

WhatsApp 一直没连上

重新执行：

openclaw channels login --channel whatsapp

9. 安全建议

不要把 .env、API Key、AppSecret、bot token 上传到公开仓库。打包发给别人时，仓库里只放示例配置，不放真实密钥。
